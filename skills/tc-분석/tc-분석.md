---

## 역할

기획서를 읽어 분석하고 analysis.md(분석_text_1.md)를 생성한다:

1. 기획서 내용 가져오기 (입력 유형에 따라 처리 방식 다름)
2. 원문을 마크다운으로 정리한 `분석_text_1.md` 작성
3. 테스트 관점 분석 (EP, BVA, 상태 전이, 결정 테이블) 수행
4. 분석 결과를 tc-설계 단계에 전달할 수 있는 상태로 저장

## 입력 유형 감지 및 처리

> ⚠️ **파이프라인 모드 우선 규칙**: 핸드오프에 `confluence_raw.md` 파일 경로가 포함되어 있으면, Confluence URL이 있더라도 **절대 `getConfluencePage`를 호출하지 않는다.** `confluence_raw.md`를 Read 도구로 읽어 그것을 기획서 원문으로 사용한다. 팀장(tc-팀-v2)이 이미 저장해둔 파일이다.

| 입력 유형        | 판별 기준                                | 처리 방식                           |
| ---------------- | ---------------------------------------- | ----------------------------------- |
| 파이프라인 파일  | 핸드오프에 `confluence_raw.md` 경로 포함 | Read 도구로 파일 직접 읽기 (최우선) |
| Confluence       | URL에 `atlassian.net` 또는 `/wiki/` 포함 | Atlassian MCP (`getConfluencePage`) |
| PDF              | `.pdf` 확장자 또는 경로                  | `Read` 도구로 직접 읽기             |
| Word (doc/docx)  | `.doc` / `.docx` 확장자                  | `Read` 도구로 직접 읽기             |
| Excel (xlsx/xls) | `.xlsx` / `.xls` 확장자                  | Bash + Node.js로 파싱               |

### Confluence 처리

> ⚠️ `confluence_raw.md`가 있으면 이 섹션 전체를 건너뛴다.

- Atlassian MCP(`getConfluencePage`)로 **ADF 형식**으로 가져오기
  - cloudId: `{{CONFLUENCE_SITE_HOST}}`
  - pageId: URL 마지막 숫자
- **취소선(`strike` mark) 포함 모든 서식 누락 없이 확인**
  - 취소선 텍스트는 반드시 `~~텍스트~~` 형식으로 변환
  - 링크, 볼드, 이탤릭 등 서식도 그대로 반영

### PDF / doc / docx 처리

1. Read 도구로 파일 읽기
2. 기능명: 파일명에서 추출 (확장자 제거, 공백→`_`, 특수문자 제거)
3. 읽은 내용을 그대로 원문으로 사용 → 분석_text_1.md 작성

### Excel (xlsx/xls) 처리

```bash
{{NODE_PATH}} -e "
const XLSX = require('xlsx');
const wb = XLSX.readFile('[파일경로]');
wb.SheetNames.forEach(name => {
  const csv = XLSX.utils.sheet_to_csv(wb.Sheets[name]);
  console.log('=== Sheet: ' + name + ' ===');
  console.log(csv);
});
"
```

> xlsx 모듈 없을 시: `cd {{WORK_ROOT}} && npm install xlsx` 후 재실행.

## ADF 이미지 컨텍스트 분석

Confluence 이미지는 다운로드 불가. ADF `type: "media"` 노드의 메타데이터(alt, width/height, 문서 내 위치)와 **직전/직후 텍스트**로 맥락을 파악한다.

**처리 기준**:

| 조건 | 처리 |
|------|------|
| 전후 텍스트가 기능/조건/동작을 상세 기술 | 텍스트 기반 분석 (이미지 무시) |
| 전후 텍스트가 "예시", "참고"만 있음 | `[이미지 참조]` 태그 + 메타데이터 기반 추정 |
| 텍스트 없이 이미지만 단독 배치 | `[이미지 참조 필요]` + TC 작성 시 원본 확인 요청 |
| alt에 `.gif`/`.mp4` 포함 | `[연출 참조 필요]` — 실제 동작 확인 필수 |

**분석_text_1.md 반영**: 이미지 위치에 `> [이미지: {alt}] ({size}) / 위치: {섹션} / 추정: {내용}` 기록
**설계_text_2.md 반영**: 설명 부족 이미지가 있는 소분류에 `[이미지 참조 필요]` 추가
**커버리지 매핑표 반영**: 해당 항목 검증 포인트에 `[이미지 참조]` 표기

## analysis.md 필수 구조 (템플릿 강제)

> ⚠️ **CRITICAL**: analysis.md는 **3개 Part**로 구성된다. Part A만 작성하고 종료하면 자체 검증 FAIL.
> 각 Part의 모든 섹션을 **순서대로** 작성하고, 해당 없을 시 `"해당 없음"` 명시 (섹션 생략 금지).

### Part 경계 원칙 (중복 방지)

| Part | 역할 | 책임 |
|------|------|------|
| **Part A** | 원문 정리본 | 기획서 내용을 마크다운으로 보존 |
| **Part B** | 테스트 분석 레이어 | **"후보 식별"** — EP/BVA/상태/결정/태그 후보만 발견 |
| **Part C** | 설계 힌트 | 설계 단계(tc-설계.md)로 넘기는 초안/코멘트 |

**중복 방지 규칙**:
- analysis.md는 **"후보 목록"**까지만 작성 (실제 TC/매핑표/배분표는 설계 단계 책임)
- tc_design.md는 analysis.md의 후보를 받아 **"확정 테이블"**로 변환 (커버리지 매핑표, 검증단계 배분표, 상태 전이 실제 엣지, 결정 테이블 실제 케이스)
- 동일 정보를 두 파일에 복제하지 말 것. analysis = "What exists", design = "What to test"

### Part A — 원문 정리본

**A-1. 개요** — 문서 목적, 주요 포인트
**A-2. 기능 정의** — 기획서 내용을 마크다운으로 정리 (원문 최대 보존)
  - 취소선: `~~텍스트~~` 변환
  - 플랫폼별 조작: **모바일** / **PC** 볼드 표기
  - 외부 링크, 미완성 메모, 플레이스홀더 그대로 보존
**A-3. 이미지 분석 통합** — image_analysis.md 내용을 본 섹션에 인라인 병합 (섹션별로 관련 이미지 요약 인용)
**A-4. 토스트/팝업 메시지 목록** — `키 | 문구 | 발생 상황` 표
**A-5. GlobalDefine 키 목록** — `키 | 초기값 | 용도` 표
**A-6. 취소선/추후구현 항목** — 원문에서 추출 (없으면 "없음" 명시)

### Part B — 테스트 분석 레이어 (후보 식별)

> 이 Part는 **후보 목록**만 작성한다. 실제 테이블/매핑은 tc-설계.md가 담당.

**B-1. 용어 사전**
기능 고유 용어를 표로 정의. TC 문장 일관성 확보 목적.

```markdown
| 용어 | 정의 | 동의어/관련 |
|------|------|------------|
| [용어1] | [정의] | [동의어] |
```

**B-2. 외부 의존성 & 연계 기능**
본문에 등장한 외부 시스템/데이터 출처/연계 기능을 집계.

```markdown
| # | 의존 대상 | 유형 | 등장 섹션 | 관계 유형 (1:1/1:M/M:N) |
|---|---------|------|----------|---------------------|
| 1 | 길드 시스템 | 연계 기능 | A-10.3 | 1:1 |
```

**B-3. EP 파티션 후보** (EP=동등 분할 / Equivalence Partitioning)
수치/상태/입력 조건이 있는 소분류에 대해 유효/무효 파티션 식별. 유효 파티션은 정상 1건으로 대표, 무효 파티션은 **다른 이유로 실패**하는 구간마다 부정 1건 필수.

```markdown
| # | 소분류 | 입력/조건 | 유효 파티션 | 무효 파티션 (분리 사유) |
|---|--------|----------|-----------|-----------------------|
| 1 | 삭제 슬롯 수 | 보유 슬롯 | `1~3` | `0` (소진) / `> 3` (초과) |
```

**B-4. BVA 후보 수치** (BVA=경계값 분석 / Boundary Value Analysis)
수치 제한이 있는 소분류의 경계값 전수 수집. 4포인트 = `min-1`(실패) / `min`(성공) / `max`(성공) / `max+1`(실패). 실제 BVA TC는 설계 단계에서 작성.

```markdown
| # | 소분류 | 수치 출처 (GlobalDefine 키) | 현재값 | min-1 / min / max / max+1 |
|---|--------|---------------------------|--------|--------------------------|
| 1 | 최대 캐릭터 슬롯 | Define 페이지 | 5 | 0 / 1 / 5 / 6 |
```

**B-5. 상태 머신 후보 목록**
상태를 가진 소분류를 식별하고 기본 상태/트리거를 스케치.

**대상 판별 기준** — 소분류에 아래 패턴이 있으면 상태 머신 후보:
- 활성/비활성 토글 (버프 ON/OFF, 차단/해제)
- 시간 기반 전이 (쿨타임, 카운트다운, 대기 시간)
- 진행 단계 (퀘스트 미수락→진행→완료, 강화 단계)
- 자원 소모 (내구도 감소, 스택 소모/충전)
- 잠금/해제 (콘텐츠 잠금, 레벨 해금, 권한 변경)

> 실제 상태 전이 테이블(전이 엣지 전수)은 tc-설계.md (EVAL-12)에서 완성.

```markdown
| # | 소분류 | 상태 종류 | 주요 트리거 | 복잡도 |
|---|--------|----------|-----------|--------|
| 1 | 캐릭터 삭제 진행 | 없음 / 대기 / 취소 / 완료 | 삭제 신청, 시간 경과, 만료 | 중 |
```

**B-6. 결정 테이블 후보 목록**
복합 조건(2개 이상 AND/OR)이 존재하는 소분류 식별.

**대상 판별 기준** — "A이고 B이면", "A이거나 B이면", 복수 조건이 동시에 체크되는 기능.

> 실제 결정 테이블(조건 매트릭스)은 tc-설계.md (EVAL-13)에서 완성.

```markdown
| # | 소분류 | 조건 개수 | 조건 목록 | 결정 결과 유형 |
|---|--------|---------|---------|------------|
| 1 | 캐릭터 삭제 조건 | 2 | 레벨 ≥ Limit, 길드 미소속 | 성공 / 레벨 오류 / 길드 오류 / 복합 오류 |
```

**B-7. 암묵적 요구사항 태그 매핑**
**모든** 소분류를 전수 나열하고 해당 태그를 붙인다. 빈 셀 금지 — "해당 없음" 명시.

```markdown
| 소분류 | 적용 태그 | 근거 (한 줄) |
|--------|----------|------------|
| 캐릭터 삭제 확인 | [데이터] [동시성] [UI] | 재화·캐릭터 삭제, 다단계 흐름 |
| 캐릭터 선택 화면 UI | 해당 없음 | — |
```

> 태그 후보: `[세션] [권한] [데이터] [UI] [유저상태] [동시성] [대소문자] [시간조작] [연계]`

**B-8. 오류 패턴 매칭 (Error Guessing)**
각 소분류를 "오류 추정 카탈로그" (본 문서 후반)의 행과 대조하여 매칭.

```markdown
| 소분류 | 매칭 기능 유형 | 반영 필요 오류 패턴 | 비고 |
|--------|-------------|------------------|------|
| 캐릭터 삭제 진행 | 상태 전이 + 시간 기반 | 앱 강종, 네트워크 끊김, 비정상 재전이 | HIGH |
```

### Part C — 설계 힌트 (초안, 설계 단계에서 확정)

**C-1. 섹션별 리스크 초안**
설계 단계에서 `[HIGH]/[MEDIUM]/[LOW]`로 확정될 초안.

```markdown
| 섹션 | 초안 리스크 | 근거 (한 줄) |
|------|-----------|------------|
| A-10 캐릭터 삭제 | [HIGH] | 재화/캐릭터 손실, 다단계 흐름, 동시성 |
| A-6 캐릭터 모델링 | [LOW] | 단순 표시, 연출 |
```

**C-2. TC 사전 배분 초안**
설계 단계 "검증단계 사전 배분표"로 발전될 초안. 섹션별 예상 수량만 기재.

```markdown
| 섹션 | 기본기능 | QA 정상 | QA 부정 | QA 예외 | 합계 (예상) |
|------|---------|-------|--------|-------|-----------|
| A-10 캐릭터 삭제 | N | N | N | N | N |
```
> 수량은 기획서 복잡도에 따라 자유롭게. 고정 수량 없음.

**C-3. 미지정값 재확인 체크리스트**
기획서에서 `N (미지정)`, `TBD`, `시간 (미지정)` 등으로 남은 값을 전수 추출. 기획자 확인 필요 플래그.

```markdown
| # | 키/항목 | 등장 위치 | 현재 상태 | 재확인 필요 여부 |
|---|--------|----------|---------|---------------|
| 1 | Character_DeleteLevelLimit | A-10.3 | N (미지정) | ⚠️ 필수 |
```

## 오류 추정 카탈로그 (Error Guessing)

기능 유형별로 자주 누락되는 오류 패턴 목록. 설계_text_2.md 작성 시 소분류를 기능 유형에 대조하여, 해당 패턴이 이미 반영됐는지 확인한다.

| 기능 유형 | 연관 암묵적 요구사항 태그 | 자주 누락되는 오류 패턴 | 설계_text_2.md 반영 방법 |
|---------|---------------------|------------------|-------------------|
| 수량/재화 변경 | `[데이터]` `[동시성]` | 동시 요청 중복 소모, 음수/0 입력, 롤백 없는 실패, 최대치 초과 소모 | `[동시성]` 태그 + BVA 4포인트 |
| 상태 전이 | `[세션]` `[데이터]` | 중간 상태에서 앱 강제 종료, 네트워크 끊김 후 상태 불일치, 동일 이벤트 재발생(비정상 전이) | 상태 전이 테이블 작성 + 예외 행 포함 |
| 시간 기반 기능 | `[시간조작]` `[세션]` | 클라이언트 시간 조작, 자정/주간 갱신 경계, 앱 백그라운드 중 시간 경과 | `[시간조작]` 태그 + 경계 시점 TC 명시 |
| 조건부 잠금/해제 | `[권한]` `[연계]` | 조건 변경 직후 캐시 미반영, 복수 조건 중 하나만 충족, 조건 충족 후 재잠금 여부 | 결정 테이블 작성 + `[권한]` 태그 |
| 목록/검색/페이지네이션 | `[UI]` `[데이터]` | 빈 목록(0건), 최대 항목 수 도달, 마지막 페이지 경계, 정렬 기준 변경 후 유지 여부 | NULL/빈상태 대상 목록 + BVA |
| 텍스트 입력 | `[대소문자]` `[데이터]` | 최대 길이 초과, 특수문자/이모지 입력, 공백만 입력, 복사-붙여넣기 | BVA 최대 길이 + `[대소문자]` 태그 |
| 연동/인터페이스 | `[연계]` `[데이터]` | A 기능 완료 후 B 화면 미반영, 연동 기능 삭제 후 참조 오류, 순서 의존성 역전 | 기능 간 관계 테이블 + 교차 TC 명시 |

**사용 방법**:
1. 설계_text_2.md 소분류 작성 시 위 표에서 해당 기능 유형 행을 찾아 대조
2. 오류 패턴이 소분류 설명 또는 검증단계 배분표에 반영됐는지 확인
3. 반영 안 됐으면 해당 소분류에 추가 (부정/예외 케이스로 보강)

> 암묵적 요구사항 태그와 1:1 연결됨 — 태그가 붙은 소분류는 반드시 해당 행의 오류 패턴을 검토.

## 경계값 참조표

수치 조건이 있는 기능 분석 시 아래 표를 기준으로 경계값 EP/BVA 포인트를 도출한다.

| ------------- | ---------------------------------------------------- |
| 마나/MP       | 소모량-1 (부정) / 정확히 소모량 (예외)               |
| HP            | 1 생존 최소 / 0 사망 / 최대HP                        |
| 골드/재화     | 구매가-1 (부정) / 정확히 구매가 (예외) / 최대 보유량 |

| 스택/중첩     | 최대-1 / 최대 / 최대+1 시도                          |
| 쿨타임        | 쿨타임 종료 직전 / 정확히 0                          |
