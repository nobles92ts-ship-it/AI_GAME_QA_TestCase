# TC 수정 규격

> **목적**: 리뷰 보고서에 작성된 `처방:` 지시문을 그대로 실행한다. tc-fixer는 "어떻게 고칠지" 판단하지 않는다. 판단은 리뷰어(qa-reviewer)가 한다.

**⚠️ 처방 없는 이슈는 실행하지 않는다**: 리뷰 보고서의 이슈에 `처방:` 필드가 없으면, 해당 이슈는 스킵하고 보고서에 `[처방 없음 — 리뷰어 확인 필요]`로 기록한다. 임의로 수정 방향을 결정하지 않는다.

**⚠️ TC 시트 입력 시 반드시 tc-생성 스킬 기준 적용:**
실제 TC를 시트에 입력(신규 추가, 값 수정, 재작성)할 때는 `{CLAUDE_HOME}\tc-team-v2\skills\tc-생성\tc-생성.md`의 서식 규칙을 전체 적용할 것.

**⚠️ 신규 TC 삽입 시 컬럼 순서 엄수 (과거 발생 버그):**
배열 순서 반드시 `["대분류","중분류","소분류","검증단계","재현스탭","플랫폼","비고"]` — F열(index 4)=재현스탭, G열(index 5)=플랫폼.
반대로 입력하면 시트에서 F열=플랫폼/G열=긴 한글 문장으로 뒤바뀐다. (2026-04-17 sheet2 TC 176~196에서 재발)

## 처방 실행 순서

**CRITICAL → HIGH → MEDIUM → LOW 순으로 처방을 읽고 순서대로 실행:**

| 처방 유형 | 실행 방법 |
|----------|----------|
| `F열 수정` (재현스탭) | 처방의 초안을 Haiku에 전달 → 완성 문장으로 변환 → 시트 적용 |
| `E열 수정` (검증단계) | 처방값 그대로 적용 |
| `G열 수정` (플랫폼) | 처방값 그대로 적용 |
| `J열 추가` (비고) | 처방값 그대로 적용 |
| `신규 TC 추가` | 처방의 초안을 Haiku에 전달 → 재현스탭 완성 → 적절한 위치에 삽입 |
| `삭제` | 해당 TC 행 삭제 |

### Haiku 역할 (재현스탭 한정)

처방에 재현스탭 초안이 있는 경우만 Haiku를 호출한다:
- 입력: 처방의 `'[초안]'` 문장
- 출력: `[사전조건]+[행동]+[기대결과]` 완성 문장 1줄
- 판단 없음: 소분류, 검증단계, 플랫폼은 처방값 그대로

## 이슈 심각도 참조

심각도는 처리 순서 결정에만 사용한다. 수정 내용은 처방을 따른다.
- **CRITICAL**: 먼저 처리
- **HIGH**: 그 다음
- **MEDIUM**: 그 다음
- **LOW**: 마지막

## 수정 유형별 처리

### 소분류 누락
해당 TC 소분류 열에 값 추가

### 검증단계 오류
부정→예외, 예외→정상 등 올바른 값으로 수정

### 재현 스탭 수정
불명확한 재현 스탭을 구체화. **"정상 동작한다", "정상적으로 작동한다" 등 추상적 표현 절대 금지** → 반드시 구체적 기대 결과(화면 변화, 수치 변화, 상태 변화)를 명시

### 비고 추가
기획서 미명시, 추론 기반, HOLD 등 기재

### 중복 TC 처리
- 완전 중복 → 삭제
- PC/모바일 분리 중복 → "PC/모바일"로 병합 (PC결과=미진행, 모바일결과=미진행)

## 신규 TC 삽입 규칙 (필수)

**절대 시트 끝이나 기존 TC 하위 리스트에 추가하지 않는다.**
**동일 분류가 시트 내 여러 곳에 흩어지면 안 된다.**

신규 TC는 반드시 해당 **분류(대분류/중분류/소분류)에 맞는 위치**에 삽입:

1. 시트 데이터를 읽은 뒤, 각 행의 실제 분류를 파악 (빈 셀은 위쪽 행에서 상속)
2. 신규 TC의 대분류/중분류/소분류와 일치하는 그룹의 **마지막 행 바로 뒤**에 삽입
3. 해당 분류 그룹이 없으면, 대분류 그룹의 마지막 행 뒤에 삽입
4. 삽입 후 TC ID를 001부터 순서대로 재부여
5. 대분류/중분류/소분류 중복 표기를 정리 (최상단 1회만 표기)

### 신규 TC 필수 적용 사항
- 플랫폼별 결과 열 자동 설정: PC→모바일 N/A, 모바일→PC N/A
- 해당 결과 열은 '미진행'으로 초기화
- 검증단계: 정상 / 부정 / 예외 중 하나
- 재현 스탭: 구체적이고 검증 가능한 한 줄 문장
- **비고(J열)**: 필수 필드 — 기획서 미명시/추론 기반은 메모 기재, 해당 없으면 **빈 문자열(`""`)로 명시적 설정** (누락 시 undefined 발생)

### 신규 TC 처방 포맷 (필수 필드 6개)

리뷰어가 `처방: [소분류] 소분류에 신규 TC 추가` 처방을 작성할 때 **아래 6필드 모두** 명시해야 fixer가 판단 없이 실행 가능:

```
처방: [소분류명] 소분류에 신규 TC 추가
- 검증단계: 정상 | 부정 | 예외
- 플랫폼: PC | 모바일 | PC/모바일
- 재현스탭 초안: [사전조건]에서 [행동]하면 [기대결과]가 되는지 확인
- 비고: [메모 또는 "" 빈 문자열]
- 삽입 위치: [소분류] 그룹 마지막 뒤 (기본) | [TC ID] 뒤 (명시 시)
- 근거: [EVAL-XX 또는 분석.md 섹션]
```

> ⚠️ 6필드 중 하나라도 누락되면 fixer는 `[처방 불완전 — 리뷰어 확인 필요]`로 기록하고 스킵.

## 최종 정리 (필수, 모든 수정/삽입 후 실행)

1. **분류별 그룹핑** (`groupByCategory`): 분산된 동일 분류를 하나로 합침
2. **분류 중복 표기 제거** (`renumberAndDedup`): 이전 행과 같으면 빈칸 (최상단 1회만 표기)
3. **TC ID 재부여**: 001, 002… 3자리 숫자

## 취소선 교차 검증 (Confluence 기획서 원본 대조)

Atlassian MCP 도구(`getConfluencePage`, contentFormat: `adf`)로 원본 Confluence 기획서를 읽어 **취소선(`"type":"strike"`) 처리된 텍스트**를 모두 추출.

추출한 취소선 목록과 현재 TC 시트를 대조하여:
- 취소선 처리된 기능/스펙 기반으로 작성된 TC가 있으면 **삭제 대상**으로 표시
- 취소선 뒤에 변경된 내용(→ 이후 텍스트)이 있으면, TC가 변경 후 내용을 반영하고 있는지 확인
- 취소선 내용이 TC에 반영되어 있지 않으면 정상 → SKIP

보고서에 **[취소선 검증]** 섹션을 추가하여 결과를 기록.

## 서식 일괄 적용 (tc-생성 스펙 준수, 필수)

모든 수정/삽입 후 반드시 서식을 재적용:
- 데이터 행 전체: 흰색 배경(#FFFFFF), 검정 텍스트(#000000), bold 없음, 가운데 정렬
- 재현 스탭(F열): 왼쪽 정렬 + 자동 줄바꿈(WRAP)
- 검증단계(E열) 조건부 서식: 정상=연초록(#D9EBD3) / 부정=연빨강(#EACBCB) / 예외=연노랑(#FFF2CC)
- 결과 열(H/I) 조건부 서식: PASS=파랑(#2173DB)+흰글씨 / FAIL=빨강(#C00000)+흰글씨 / BLOCK=검정+흰글씨 / N/A=옅은회색(#EFEFEF)+검정글씨
- 드롭다운: 검증단계(정상/부정/예외), 플랫폼(PC/모바일/PC모바일), 결과 2열(미진행/PASS/FAIL/BLOCK/N/A)
- 필터 재적용: 전체 데이터 범위에 BasicFilter 설정

## 수정 완료 검증 (처방 전수 대조)

모든 처방 실행 후, 보고 전 아래를 반드시 확인:

1. **처방 전수 대조**: 리뷰 보고서의 모든 `처방:` 항목을 나열하고, 각각 실행 완료 여부를 DONE/SKIP(처방 없음)/FAIL로 표기
2. **FAIL 항목 재시도**: FAIL이 있으면 원인 파악 후 재실행. 2회 실패 시 보고서에 `[수정 실패 — 수동 확인 필요]` 표기
3. **#ERROR! 잔류 검사**: 수정 후 시트 전체에서 `#ERROR!` 값 검색 — 0건이어야 통과
4. **분류 분산 검사**: 수정으로 신규 TC 삽입 시, 동일 소분류가 연속 배치되어 있는지 확인

> **학습 근거**: P-09(1차 수정 미반영) — 월드맵 7건 중 6건 미반영 사례

## Node.js 유틸리티 함수

수정 스크립트에서 사용하는 공통 함수 (`tc_utilities.js`):
- `resolveCategories(rows)` — 빈 셀을 위쪽 행에서 상속하여 실제 분류 파악
- `findInsertIndex(categories, newTc)` — 신규 TC 삽입 위치 계산
- `groupByCategory(rows)` — 분산된 동일 분류를 하나로 합침
- `renumberAndDedup(rows)` — TC ID 재부여 + 분류 중복 표기 정리
- `addResultCondFormat(sheetId, colIndex, rowCount)` — 결과 열 조건부 서식
- `addVerifCondFormat(sheetId, rowCount)` — 검증단계 조건부 서식
