# 완료처리 스킬

TC 파이프라인 완료 후 3단계 순차 실행.

> ⚠️ **용어 주의**: 내부 단계 `FINAL-1 / FINAL-2 / FINAL-3`. tc-팀-v2 파이프라인 STEP 1~7과 혼동 금지.

---

## 설정

```
NODE  = {NODE_PATH}
UTIL  = {WORK_ROOT}/scripts/util
SPECS = {WORK_ROOT}/team/specs
```

---

## 트리거

- `/완료처리 [기능명]`
- "완료처리 [기능명]" / "완료처리 해줘 [기능명]"
- tc-팀-v2 완료 시 자동 호출

---

## 입력

| 항목 | 소스 |
|------|------|
| 기능명 | 사용자 입력 or 호출 컨텍스트 |
| SHEET_ID / TAB_NAME / CONFLUENCE_URL | `$SPECS/[기능명]/sheet_info.txt` |

sheet_info.txt 형식:
```
SHEET_ID=...
TAB_NAME=...
CONFLUENCE_URL=...
```

---

## 실행 순서

### FINAL-1: 대시보드 업데이트

```bash
cd "$UTIL" && "$NODE" update_dashboard.js "$SHEET_ID"
```

- 성공 → FINAL-2 / 실패 → 오류 출력 후 FINAL-2~3 계속

---

### FINAL-2: K~O열 프로젝트 정보 + 미니 결과 대시보드

```bash
"$NODE" "$UTIL/add_project_info.js" "$SHEET_ID" "$TAB_NAME" "$CONFLUENCE_URL"
```

**K/L열 고정 패널 (1~10행)**
- K1:L1 "담당자" 헤더 (병합, 남색)
- K2~K5: "기획"/"서버"/"클라"/"UI" | L2~L5: 빈칸 (담당자 이름 직접 입력)
- K6: "기획서" | L6: Confluence URL 하이퍼링크
- K7 "테스트 환경" 헤더 (남색, 병합 없음) | L7 빈칸
- K8~K10: "날짜/시간"/"클라이언트"/"서버" | L8~L10: 빈칸 (테스트 환경 정보 직접 입력)

**M~O열 미니 결과 대시보드 (1~7행)**
- M1/N1/O1: "구분"/"PC"/"모바일" 헤더
- M2~M6: PASS/FAIL/BLOCK/미진행/N/A (텍스트 라벨, 메트릭별 색상 구분)
- M7: "합계"
- N/O 열: H열(PC)·I열(모바일) 결과 COUNTIF 수식 자동 집계, 합계는 SUMPRODUCT
- 대시보드(`update_dashboard.js`)와 동일한 PC=H / 모바일=I 컬럼 규칙 따름

- 성공 → FINAL-3 / 실패 → 오류 출력 후 계속

---

### FINAL-3: 드라이브 전체 sync

```bash
"$NODE" "$UTIL/upload_md_to_drive.js" --sync "$FEATURE_NAME"
```

- specs/[기능명]/ 폴더 전체 드라이브 업로드
- 성공 → 완료 보고 / 실패 → 오류 출력

### FINAL-4: 리포트 수치 재덤프 검증 (필수) ⚠

리포트(요약 보고/AB 비교/QA 결과)에 기재할 수치는 **반드시 시트 재덤프 → 실측 계산값** 사용. step_result.json의 중간 집계나 이전 검증 시점 수치 그대로 옮겨 적기 금지.

```bash
"$NODE" "$UTIL/read_gsheet_data.js" "$SHEET_ID" "$TAB_NAME" \
  > "$SPECS/$FEATURE_NAME/_final_dump.json"
```

재계산 필수 지표 (리포트 기재 직전):
- TC 총 수 / QA TC 수 (기본기능 제외)
- 검증단계 분포 (정상/부정/예외) + 부정+예외 비율
- 추상 표현 검출 건수 (F열 grep)
- `#ERROR!` / `Output Format:` 잔류 0건 확인
- G열 컬럼 꼬임 0건 (플랫폼 외 값 검출 없음)
- J열 화이트리스트 위반 0건

**검증 실패(이전 기재 수치와 실측 불일치) 시**: 리포트 기재값 즉시 정정 + `_final_dump.json` 경로를 리포트 각주에 링크.

> 2026-04-22 회귀 사례: AB 비교 리포트에 "A 추상표현 0건" 기재했으나 실측 1건. 재덤프 미실행이 원인.

## 주의사항

- 3단계 순서대로 실행 (병렬 금지)
- 한 단계 실패해도 나머지 계속 실행
- sheet_info.txt 없으면 사용자에게 기능명/파라미터 직접 요청
