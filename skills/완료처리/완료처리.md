# 완료처리 스킬 (백업 사본)

> ⚠️ **이 파일은 백업용입니다.** 실제 SSoT는
> `~/.claude/tc-team-v2/skills/완료처리/완료처리.md` 입니다.
> tc-팀-v2 파이프라인은 SSoT만 참조합니다. 이 파일을 수정하더라도 파이프라인에 반영되지 않습니다.
> 드리프트 방지를 위해 변경이 필요하면 SSoT를 먼저 갱신하세요.

tc-팀-v2 파이프라인이 STEP 1~7(설계 → 2차 리뷰+수정)을 완주한 직후 호출된다.
이 스킬은 **파이프라인 종료 마무리 3단계**를 순서대로 실행한다.

> ⚠️ 혼동 주의: 여기서 말하는 STEP 1/2/3은 **완료처리 스킬 내부 단계**다.
> tc-팀-v2 파이프라인의 STEP 1(설계)과는 완전히 별개.

---

## 호출자가 보장해야 하는 환경 변수

tc-팀-v2 에이전트가 이 스킬을 실행하기 전 아래 값을 쉘 환경 또는 치환 형태로 제공해야 한다.

| 변수 | 의미 | 출처 |
|------|------|------|
| `NODE` | node.exe 절대경로 | `{NODE_PATH}` |
| `UTIL` | 유틸 스크립트 폴더 | `{WORK_ROOT}/scripts/util` |
| `SHEET_ID` | 대상 스프레드시트 ID | 파이프라인 초기화 시 파싱 |
| `TAB_NAME` | 시트 탭명 (공백 포함 가능) | `sheet_info.txt` |
| `CONFLUENCE_URL` | 원본 Confluence URL | `sheet_info.txt` |
| `FEATURE_NAME` | 기능명 (폴더명) | 파이프라인 초기화 시 결정 |

> 값이 비어 있으면 해당 STEP은 **중단하고 오류 보고** — 임의 기본값 사용 금지.

## STEP 1 — 대시보드 업데이트

목적: 마스터 대시보드(DX 전체)에 신규/갱신된 TC 시트 반영.

**실행 명령**
```bash
cd "$UTIL" && "$NODE" update_dashboard.js "$SHEET_ID"
```

**규칙·수식·빈칸 처리 등 세부 SSoT**
→ [skills/tc-대시보드/TC-Dashboard.md](../tc-대시보드/TC-Dashboard.md)

이 스킬은 대시보드 갱신 "실행"만 담당한다. 수식 변경, 새 대시보드 추가, 빈칸 유지 규칙 등은
**tc-대시보드 스킬 문서가 단일 SSoT**다. 완료처리에서 임의 수정하지 말 것.

**오류 처리**
- exit code ≠ 0 → 1회 재시도 → 그래도 실패 시 STEP 2/3 진행하지 말고 중단, 사용자에게 보고
- stdout에 `OAuth` 또는 `token` 키워드 → 토큰 만료 가능성, 재인증 안내 후 중단

---

## STEP 2 — K/L열 프로젝트 정보 패널 추가

목적: TC 시트 우측(K/L열)에 Confluence 링크·기능명 등 프로젝트 메타 패널 삽입.

**실행 명령**
```bash
"$NODE" "$UTIL/add_project_info.js" "$SHEET_ID" "$TAB_NAME" "$CONFLUENCE_URL"
```

**필수 인자 검증**
- `$TAB_NAME`이 공백을 포함할 수 있으므로 반드시 쌍따옴표 유지
- 3개 인자 모두 비어 있지 않은지 실행 전 확인

**오류 처리**
- exit code ≠ 0 → 1회 재시도 → 그래도 실패 시 STEP 3 진행하지 말고 중단
- "TAB not found" 계열 오류 → `sheet_info.txt`의 TAB_NAME 재확인 지시

---

## STEP 3 — 드라이브 sync

목적: specs 폴더의 기획서·분석·설계 MD 파일을 구글 드라이브 기획서 폴더로 동기화.

**실행 명령**
```bash
"$NODE" "$UTIL/upload_md_to_drive.js" --sync "$FEATURE_NAME"
```

**오류 처리**
- exit code ≠ 0 → 1회 재시도 → 실패 시 중단 보고 (STEP 1/2는 성공 상태 유지)
- 드라이브 API 쿼터 오류 → 60초 대기 후 1회 재시도

## 완료 보고 (호출자에게 반환)

세 단계 완료 후 tc-팀-v2 팀장에게 아래 형식으로 결과 반환:

```
완료처리: 대시보드 ✓ / K·L패널 ✓ / 드라이브 sync ✓
```

실패 시 실패 단계까지만 체크 표시:
```
완료처리: 대시보드 ✓ / K·L패널 ✗ (사유) / 드라이브 sync ⏸
```

> `⏸` = 이전 단계 실패로 미실행. 부분 성공 상태를 명확히 구분한다.

## 주의사항

- **3단계는 직렬 실행**. 병렬화 금지 (드라이브 sync가 대시보드 결과를 참조할 수 있음).
- **중간 단계 실패 시 뒤 단계 실행 금지** — 사용자가 원인 확인 후 재개해야 함.
- 이 스킬은 **읽기·실행 전용**. TC 데이터 시트를 직접 수정하지 않는다.
- 재시도는 각 STEP당 **최대 1회**. 무한 재시도 금지.
