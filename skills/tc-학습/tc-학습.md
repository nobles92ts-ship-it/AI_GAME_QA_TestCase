# TC 학습 피드백 DB

> **목적**: 리뷰/QA에서 반복 발견된 이슈를 패턴화하여 동일 실수 재발을 사전 차단한다.
> **읽는 주체**: tc-설계검수 (설계 검증 + writer 전달 지시 생성), qa-reviewer (패턴 축적)
> **읽지 않는 주체**: tc-designer (창작 단계), tc-writer (코더), tc-fixer (코더)

모든 내용은 **한국어**로 작성한다.

---

## 설계 패턴

> tc-설계검수가 설계물(tc_design.md)을 이 패턴들로 검증한다.
> 이슈 발견 시 설계검수 보고서에 기재 → designer가 설계 보완.

### P-11: 기획서 "규칙/기조" 섹션이 TC로 변환 안 됨

- **증상**: "~하지 않는다", "~금지", "~반드시 포함" 등으로 기술된 항목이 분류 트리에서 부정 TC로 변환되지 않음
- **검수**: 기획서 내 규칙/기조/금지 표현 섹션이 분류 트리에 대응하는 부정 TC로 포함되어 있는지

### P-12: BVA가 max 1건만 설계되고 경계 직전(max-1) 누락

- **증상**: 시간/수량 경계값에서 max(종료) 1건만 있고 max-1(유지 확인) TC가 없음
- **검수**: 시간/수량 제한이 있는 소분류 배분표에 max-1(유지) + max(종료) 쌍이 있는지

### P-05: 교차 중복 TC `규칙파일 반영됨 — 재발 감시`

- **검수**: 분류 트리 내 동일 트리거가 2곳 이상이면 통합 또는 범위 분리 확인

### P-08: 상태 전이 엣지 미커버 `규칙파일 반영됨 — 재발 감시`

- **검수**: 상태 전이 테이블의 모든 엣지가 배분표 케이스에 1:1 매핑되어 있는지

## 작성 패턴

> tc-설계검수가 이 패턴들을 읽고, writer에게 전달할 **구체적 지시 항목**으로 변환한다.
> 설계검수 보고서의 "writer 전달 지시" 섹션에 기재 → writer가 그대로 실행.

### P-01: Output Format 템플릿 잔류 `규칙파일 반영됨 — 재발 감시`

- **지시 변환**: "F열에 'Output Format:' 문자열이 남아 있으면 완성 문장으로 교체할 것"

### P-02: 1NF 위반 — 복합 기대결과 `규칙파일 반영됨 — 재발 감시`

- **지시 변환**: "F열에 '~되고 ~되는지' 패턴이 있으면 TC 분리할 것"

### P-03: 분류 분산 배치 `규칙파일 반영됨 — 재발 감시`

- **지시 변환**: "추가 케이스는 시트 끝에 append 금지 — 기존 그룹 내 삽입할 것"

### P-04: #ERROR! 수식 오류 `규칙파일 반영됨 — 재발 감시`

- **지시 변환**: "수식 사용 금지 — JSON 직접값만 입력할 것"

### P-09: 처방 미반영 `규칙파일 반영됨 — 재발 감시`

- **지시 변환**: "수정 완료 후 처방 전수 대조 — 모든 처방 DONE/SKIP/FAIL 표기할 것"
